### 4. Семантические действия для генерации ОПС

Генерация кода происходит во время синтаксического разбора (метод рекурсивного спуска или магазинный автомат). Для каждого правила грамматики определены действия, добавляющие инструкции в выходной поток ОПС.

**Обозначения:**
*   `EMIT(OP)` — добавить инструкцию с операцией `OP`.
*   `EMIT(OP, ARG)` — добавить инструкцию с операцией `OP` и аргументом `ARG` (имя переменной, значение числа).
*   `NEW_LABEL()` — создать новую уникальную метку для переходов.
*   `BACKPATCH(LABEL, ADDR)` — заполнить адрес перехода для ранее созданной метки.

#### Таблица семантических действий

| Нетерминал | Правило в НФГ | Семантические действия |
|:---|:---|:---|
| **S** (Программа) | `S → T_ID = E ; S` | 1. Код для `E`<br>2. `EMIT(POP_VAR, T_ID.value)`<br>3. Код для следующего `S` |
| | `S → IF E THEN S ElsePart END ; S` | 1. Код для `E`<br>2. `L1 = NEW_LABEL()`<br>3. `EMIT(JMP_FALSE, L1)`<br>4. Код для `THEN` части<br>5. `L2 = NEW_LABEL()`<br>6. `EMIT(JMP, L2)`<br>7. `BACKPATCH(L1, current_addr)`<br>8. Код для `ElsePart`<br>9. `BACKPATCH(L2, current_addr)`<br>10. Код для следующего `S` |
| | `S → WHILE E DO S END ; S` | 1. `L1 = NEW_LABEL()` (начало цикла)<br>2. `BACKPATCH(L1, current_addr)`<br>3. Код для `E`<br>4. `L2 = NEW_LABEL()`<br>5. `EMIT(JMP_FALSE, L2)`<br>6. Код для тела `DO`<br>7. `EMIT(JMP, L1)`<br>8. `BACKPATCH(L2, current_addr)`<br>9. Код для следующего `S` |

### 5. Список операций ОПС

Интерпретатор поддерживает следующий набор инструкций. Все операции работают со стеком данных.

#### Арифметические и логические операции
| Код операции | Мнемоника | Описание | Стек (до → после) |
|:---|:---|:---|:---|
| 1 | `OP_ADD` | Сложение | `a, b` → `a+b` |
| 2 | `OP_SUB` | Вычитание | `a, b` → `a-b` |
| 3 | `OP_MUL` | Умножение | `a, b` → `a*b` |
| 4 | `OP_DIV` | Деление | `a, b` → `a/b` |
| 5 | `OP_NEG` | Унарный минус | `a` → `-a` |
| 6 | `OP_LT` | Меньше | `a, b` → `a < b` (1 или 0) |
| 7 | `OP_GT` | Больше | `a, b` → `a > b` (1 или 0) |
| 8 | `OP_LE` | Меньше или равно | `a, b` → `a <= b` (1 или 0) |
| 9 | `OP_GE` | Больше или равно | `a, b` → `a >= b` (1 или 0) |
| 10 | `OP_EQ` | Равно | `a, b` → `a == b` (1 или 0) |
| 11 | `OP_NE` | Не равно | `a, b` → `a != b` (1 или 0) |

#### Работа со стеком и переменными
| Код операции | Мнемоника | Описание |
|:---|:---|:---|
| 20 | `OP_PUSH_INT` | Поместить целое число на стек (аргумент: значение) |
| 21 | `OP_PUSH_FLOAT` | Поместить вещественное число на стек (аргумент: значение) |
| 22 | `OP_PUSH_VAR` | Поместить значение переменной на стек (аргумент: имя) |
| 23 | `OP_POP_VAR` | Снять значение со стека и записать в переменную (аргумент: имя) |
| 24 | `OP_INT_TO_FLOAT` | Преобразование целого в вещественное | int → float |
| 25 | `OP_FLOAT_TO_INT` | Преобразование вещественного в целое (округление) | float → int |

#### Работа с массивами
| Код операции | Мнемоника | Описание | Стек (до → после) |
|:---|:---|:---|:---|
| 30 | `OP_ARRAY_GET` | Получить элемент массива | `array_name, index` → `value` |
| 31 | `OP_ARRAY_SET` | Записать значение в элемент массива | `value, index` → – `(аргумент: имя массива)` |
| 30 | `OP_ARRAY_GET` | Получить элемент массива | `index` → `value` (аргумент: имя массива) |

#### Математические функции
| Код операции | Мнемоника | Описание | Стек (до → после) |
|:---|:---|:---|:---|
| 40 | `OP_CALL_SQRT` | Квадратный корень | `x` → `sqrt(x)` |
| 41 | `OP_CALL_EXP` | Экспонента | `x` → `exp(x)` |
| 42 | `OP_CALL_LOG` | Натуральный логарифм | `x` → `log(x)` |

#### Управление потоком
| Код операции | Мнемоника | Описание |

### 6. Формат ОПС

Программа в коде ОПС представляет собой вектор инструкций. Каждая инструкция имеет фиксированную структуру.

**Структура инструкции (C++):**

```cpp
enum OpType {
    OP_ADD, OP_SUB, OP_MUL, OP_DIV, OP_NEG,
    OP_LT, OP_GT, OP_LE, OP_GE, OP_EQ, OP_NE,
    OP_PUSH_INT, OP_PUSH_FLOAT, OP_PUSH_VAR, OP_POP_VAR,
    OP_ARRAY_GET,
    OP_CALL_SQRT, OP_CALL_EXP, OP_CALL_LOG,
    OP_JMP, OP_JMP_FALSE,
    OP_READ, OP_PRINT,
    OP_END
};

struct Instruction {
    OpType op;
    enum ArgType { NONE, STRING, INT, FLOAT, LABEL } arg_type;
    union {
        std::string str_val;
        int int_val;
        double float_val;
        int label_val;
    };
    // + конструкторы для каждого типа
};
    // Конструктор для операций без аргументов
    Instruction(OpType _op) : op(_op), arg(""), label(-1) {}
    
    // Конструктор для операций с аргументом (число, имя)
    Instruction(OpType _op, const std::string& _arg) : op(_op), arg(_arg), label(-1) {}
    
    // Конструктор для переходов
    Instruction(OpType _op, int _label) : op(_op), arg(""), label(_label) {}
};
```

**Пример генерации ОПС для выражения `a = SQRT(x + 1);`:**

1.  `PUSH_VAR x`
2.  `PUSH_INT 1`
3.  `ADD`
4.  `CALL_SQRT`
5.  `POP_VAR a`
